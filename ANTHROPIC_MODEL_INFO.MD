# Anthropic API 模型信息

**信息更新时间：2026年4月1日**

## 重要说明

**关于输入和输出限制**：
- **上下文窗口（Context Window）**：输入 tokens + 输出 tokens 的总和上限
- **最大输入**：理论上可以使用接近上下文窗口大小的输入，但需要为输出预留空间
- **最大输出**：单次响应可生成的最大 token 数量
- **实际可用输入** ≈ 上下文窗口 - 预期输出大小

例如：Claude Opus 4.6 的 1M 上下文窗口意味着输入+输出总和不超过 1M tokens。如果需要 128K 输出，则输入最多约 872K tokens。

### 关键特性说明

- **上下文窗口与输入输出**：上下文窗口 = 输入 tokens + 输出 tokens 的总和。实际可用输入需要为输出预留空间
- **Claude 4.6 系列**：最新一代模型，支持 1M 上下文（已正式可用，2026年3月）和自适应思考
- **Claude 4.5 系列**：过渡版本，Haiku 4.5 是当前最快速且成本最优的模型
- **视觉能力**：Claude 3+ 系列支持图像输入和分析
- **工具使用**：Claude 3+ 支持 Function Calling
- **扩展输出**：
  - Opus 4.6: 最多 128K tokens 输出
  - Sonnet 4.6 及所有 4.5/3.5 系列: 最多 64K tokens 输出
  - Claude 3 系列及传统模型: 最多 4K tokens 输出
- **计算机使用**：Claude 4.6 系列在 Computer Use 能力上有重大提升
- **多模态**：所有 Claude 3+ 模型支持文本和图像输入
- **定价优化**：2026年3月起，1M 上下文窗口不再有额外的"长上下文"费用

## 7. 使用建议

### 模型选择指南

**优先推荐 Claude 4.6 系列（最新）**：
- **Claude Opus 4.6**：最高难度任务、需要最强推理能力、多代理协作、需要超长输出（128K）
- **Claude Sonnet 4.6**：日常开发、办公任务、性价比最优、1M 上下文处理

**Claude 4.5 系列**：
- **Claude Haiku 4.5**：高频调用、实时对话、成本最优（$1/$5）
- **Claude Sonnet 4.5**：需要强推理和编码能力，但不需要 1M 上下文
- **Claude Opus 4.5**：高难度任务，但不需要 1M 上下文

**Claude 3.5 系列**：
- **Claude 3.5 Sonnet**：需要强推理和编码能力，64K 输出
- **Claude 3.5 Haiku**：需要快速响应和成本优化，64K 输出

**Claude 3 系列**：
- 传统版本，输出限制为 4K，建议迁移至 4.x 系列
- Claude 3 Haiku 已于 2026年4月弃用

**传统模型**：建议迁移至 Claude 3+ 或 4.x 系列

### API 调用注意事项

1. **上下文管理**：
   - 4.6 系列支持 1M 上下文（已正式可用）
   - 4.5 系列和 3.5 系列支持 200K 上下文
   - 传统模型支持 100K-200K 上下文
2. **输出限制**：
   - Opus 4.6: 最多 128K tokens
   - Sonnet 4.6 及所有 4.5/3.5 系列: 最多 64K tokens
   - Claude 3 系列及传统模型: 最多 4K tokens
   - 使用 Message Batches API 可支持最多 300K 输出（Beta）
3. **视觉输入**：仅 Claude 3+ 支持图像
4. **工具使用**：Claude 3+ 支持 Function Calling
5. **新特性**：4.6 系列支持自适应思考、上下文压缩、计算机使用
6. **速率限制**：根据账户等级不同
7. **定价变化**：2026年3月起，长上下文不再有额外费用
